位运算专题

Posted on 2021-11-12 Edited on 2021-12-05 In algorithm Views:

位运算是直接对数的二进制位进行操作，技巧性较强
本文介绍常见位操作符及技巧，并通过经典基础例题
最后顺便介绍C++标准库中bitset类

1 常用位操作符及技巧

1.1 位操作符

位操作符 功能
&（和） 同1得1
|（或） 有1得1
^（异或） 相同得0，相异得1
>>（右移） 右移，且高位补0，等价于整除2
<<（左移） 左移，且低位补0，等价于乘2

补充：

右移(>>)：5的二进制为(101)，右移后低位1抹去，高位补零，结果为(010)
(010)对应十进制数2；故5>>2等价于5/2

1.2 常用技巧

1.2.1 取二进制的第(i+1)位数

//法1：
(n>>i) & 1
//法2：
n & (1<<i)

首先，任何数&1，结果为该二进制数最低位的值
(1010)&1=0, (1011)&1=1
此处1可以理解为(0001)，在高位补零

那么，若想获取n的二进制数第i位数：

  1. 可以将第i位数右移到最低位，再&1（对应法1）；
  2. 或将1左移到第i位（对应法2）

1.2.2 二进制数的赋值

x |= (1<<i)

x一般先初始化为0
如果想将n的值赋予x，只需找到n(1010)对应值为1的位即可

一般通过循环实现
x=(0000);

x |= (1<<3);
此时x=(1000)；

x |= (1<<0);
x=(1010);

1.2.3 将最低位1置为0

n&(n-1)

n=(1010), n-1=(1001)
(1010)&(1001)=(1000);
n&(n-1)可得到n删去二进制最右侧1的结果

n&(n-1)可加速统计二进制数中1的个数，每统计完最右侧的1，即通过n&(n-1)抹去低位1，避免遍历二进制数

1.2.4 仅保留最低位的1

n&(-n)

二进制中的相反数：
将n所有二进制位取反，再将结果+1，即得到-n

示例：n=(1010), -n=(0110)
(1010)&(0110)=(0010)
结果仅保留最低位1，其余为均置0

1.2.5 异或和

x ^= num[i]

一般x初始化为0，因为0^x=x
由于两相同数进行异或的结果为0，即x xor x=0
故异或常用于找不同

leetcode简单-汉明距离
两个整数之间的 汉明距离 指的是这两个数字对应二进制位不同的位置的数目。
给你两个整数 x 和 y，计算并返回它们之间的汉明距离。

示例：
输入：x = 1, y = 4
输出：2
解释：
1 (0 0 0 1)
4 (0 1 0 0)
1和4的二进制数存在两不同数位

思路：显然借助异或和性质，异或后仅相异位会被置1
问题转化为：求解异或和的计算结果中1的个数
为统计二进制中1的个数，朴素的想法是遍历所有位
但考虑到上文提到的n&(n-1)可抹去最低位的1
我们可以令 n=n&(n-1), 每次抹去最低位的1，并递增计数器ans表示被抹去1的个数，直到n=0即可

int s=x^y; //异或后，仅不同值的位置处为1
while (s)
{
    s = (s&(s-1));
    ans++;
}

注：有个省事的函数**__builtin_popcount(i)**, 可直接计算一个32位无符号整数有多少个位为1)

2 基础经典题型

2.1 找数组中特殊的数

leetcode简单-只出现一次的数字
给定一个非空整数数组，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。

示例：
输入: [2,2,1]
输出: 1

解析：找不同多考虑异或，因为相同数异或得到0

关键：
x xor x=0
x xor 0=x

对本题中数组所有元素进行异或求和，出现两次的元素异或后会被抵消，最终只保留仅出现一次的元素。

2.2 用二进制数0、1表示状态

二进制仅包含0、1，可用于表示状态，如元素是否被选取(1表示被选取，0表示未被选取)
例如对于数组nums={1,2,3,4},可以用(0101)表示选取元素2和4

leetcode中等-子集
给你一个整数数组nums，数组中的元素互不相同。返回该数组所有可能的子集（幂集）。
解集不能包含重复的子集。你可以按任意顺序返回解集。
用二进制序列表示数值状态，并一一对应取出:

示例：
输入：nums = [1,2,3]
输出：[[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]

//核心部分
//长度为n的数组，需要长度为n的二进制码来表示每一位是否被选中
//长度为n的二进制码，对应十进制整数 1<<n
vector<int> curStep;
for (int i=0; i< (1 << n); ++i){
    curStep.clear();
    for (int j=0; j<n; ++j)
    {
        if (i & (1 << j)) {
        // i用于枚举，每次迭代仅选取i的二进制对应位为1的元素
        // j用于遍历元素每一位，判断是否为1
            curStep.push_back(nums[j]);
        }
    }
    ans.push_back(curStep);
}

补充解释：
例如nums={0,1,2,3,4,5}, 迭代到i=5=(000101)
此时j从0到5依次检查：i的二进制对应位是否为1
发现第1和第3位值为1，故此步迭代选出元素(0,2)

注意二进制低位在右侧，而nums从左侧开计数

2.3 转化思想：建立映射关系

上文提到可用二进制表示元素选取状态，显然元素不限于数字
我们通过建立映射，尝试将位运算拓展至字符串

leetcode中等-最大单词长度乘积
给定一个字符串数组words，找到length(word[i]) * length(word[j])的最大值，并且这两个单词不含有公共字母。你可以认为每个单词只包含小写字母。如果不存在这样的两个单词，返回0。

输入: [“abcw”,“baz”,“foo”,“bar”,“xtfn”,“abcdef”]
输出: 16
解释: 这两个单词为"abcw", “xtfn”。

思路：
可以令’a’=0，后续字母以’a’为偏移中心依次增加，char(‘b’)-‘a’=1, char(‘c’)-‘a’=2…
如此，实现了char与int的对应关系

接着，考虑到共26个字母，因此使用26位二进制数即可表示相应字母是否被选中
如字符串"ab", 对应二进制编码为(000…011)

最后，发现本题要求不能包含公共字母，转化为二进制数不能在同一位出现1，即a&b=0

//核心思想
for (int i=0; i<wordNums; i++)

        //对应位=1，表示该字母被选中
        states[i] |= (1<<(words[i][j] - 'a'));
    }
}
