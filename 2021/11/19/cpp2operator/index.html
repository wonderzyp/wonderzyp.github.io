重载运算符与类型转换

2021-11-19- 2021-12-05
1k- 4m-
cpp-C++

本文主要介绍几个印象深刻的点：

  1. 成员或非成员函数重载运算符的选择
  2. ++i与i++的实现细节
  3. while(cin>>i)的原理

1 重载运算符

背景：如果我们自定义一个类A，A中有两个成员mem1和mem2，我们希望将A的两个不同实例相加时，对应两个成员变量能够相加

A a1;
A a2;
A a3;
则a3=a1+a2实现为：
{
    a3.mem1=a1.mem1+a2.mem1;
    a3.mem2=a1.mem2+a2.mem2;
}

普通的内置加法运算不能自适应我们自定义的类A，因此我们需重新定义加法运算符

类似函数重载，重载运算符即重新定义运算符执行过程，本质上是一个函数调用

class A{
int mem1;
int mem2;

}
A::operator+(A::mem1, A::mem2)

1.1 成员与非成员的选择

使用重载运算符，本质上是一次函数调用，两者核心差异体现在参数调用上

1.1.1 非成员函数形式

由于“+”是二元运算符，使用时有两个参数(如2+3，2和3均是参数)。传递给operator+时，左数作为第一个函数参数，右数作为第二个

a1+a2;
//等价于：
operator+(a1,a2);

1.1.2 成员函数形式

成员函数形式的重载运算符，将左侧参数传给this指针

a1+a2;
//等价于
a1.operator(a2);

1.1.3 如何选择？

operator+操作俩混合类型对象时需注意：加法是对称的

string s="ha"; //string内置"+",用于连接两个字符串
string t1=s+"haha"; //正确，等价于s.operator+("haha")
string t2="haha"+s; //错误，等价于"haha".operator(s)

由于operator+重载形式为成员函数，调用时需将左侧参数绑定至this指针
而“haha”是const char类型，内部不具有operator+的成员函数

使用重载运算符实质上是一次函数调用，而成员或非成员函数形式的差异，在于第一个参数的绑定方式

2 前置与后置递增

++i和i++使用上的具体差异已众所周知，是什么导致它们的返回值不同呢？

2.1 代码复用思想

程序设计时，如果有某个已存在的函数可实现现阶段的需求
那么理想的方式是直接调用该函数
实际上，后置递增i++复用前置递增++i，即在定义i++的时候，调用已定义的++i

2.2 实现细节

A& A::operator++()
{
    //异常处理
    ++curr; //此处++调用内置版本
    return *this
}

A A::operator++(int)
{
    A ret=*this;
    ++*this; //调用自定义类A内部的++，同时进行异常检查
    return ret; //返回副本
}
  1. 定义后置operator++()时，存在一个不会被使用的int参数，仅为区分前置后置
  2. 后置调用前置++代码，且返回的是执行递增前的副本

3 while(cin>>i)

我第一次见到这个表达式就很好奇，为什么cin>>i能放在while条件语句中，并可读取全部的输入数据。

3.1 类型转换运算符

构造函数将算术类型的值转化为类，类型转换运算符将类转化为算数类型

class A{
public:
    A(int i){...};
    operator int() const {return val;};
private:
    std::size_t val;
}

A a(1); 将int型参数变为自定义的class A
a+1; 此时调用类型转换运算符，将a转为int类型

3.2 显式类型转换

一般较少提供类型转换，但定义向bool类型的转换较为常见
考虑一种情况：

int i=2;
cin<<i;

cin不含<<运算符，此处转化为bool类型，等价于将bool类型左移2位，导致意外情况，借助显式类型转换即可

explicit operator int() const {return val;};
//使用时：
static_cast<int>(a)+3;

编译器一般不会将显式类型转换运算符用于隐式类型转换

那么，回到最初的问题，显然cin>>i中cin被内置的类型转换函数转换为bool类型，才能用于条件表达式中
