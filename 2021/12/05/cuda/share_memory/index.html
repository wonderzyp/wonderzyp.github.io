CUDA-共享内存总结

Posted on 2021-12-05 In HPC Views:

存储体冲突会导致性能下降，填充可有效避免冲突，并提升性能

1 共享内存

GPU存在两种类型的存储：板载内存和片上内存。
全局内存属于板载内存，延迟大；共享内存属于片上内存，延迟低带宽大
使用共享内存，旨在降低访问全局内存的次数，避免较大的延迟，提升性能

2 存储体冲突与填充

2.1 存储体冲突

共享内存被划分为32个存储体（对应一个线程束内存在32个线程），它们可以被同时访问。当多个地址请求落在同一个存储体中，会发生存储体冲突。

如上图所示，Bank0~31表示32个存储体，若同一时刻有两个线程分别访问Bank0下的地址（序号0、32、64、96），就会发生存储体冲突，导致性能降低。
理想状态下，同一时刻应访问序号0至31。由于0至31属于不同的存储体，因此不会产生存储体冲突，且能够充分利用带宽。

2.2 填充以避免冲突

当需要按列读取元素时，由于每一列隶属于同一个存储体，因此会产生存储体冲突。借助填充手段，可以将原先属于同一存储体的列元素，打乱到不同存储体中，从而避免存储体冲突。

为节省篇幅，以5个存储体为例（实际上为32个）。 当按列读取第一列时，所有元素均在存储体Bank0中，发生存储体冲突。 假想在末列填充一列占位内存，等价于每隔5个元素填充一次，则填充后的存储体结构如下图所示：

注意：

  1. 填充是我们假想在原有最后一个存储体后增加一列，实际上存储体数量是固定的（32，本文假设为5）
  2. 填充的内存不存储元素，只起到占位符的作用，以调整原始元素的位置

3 两种填充方式：静态与动态

下文为实现矩阵转置的关键代码：对于一个存储的二维矩阵，按列读取即可实现矩阵转置
尝试两种填充方法，避免因按列读取产生的存储体冲突

3.1 静态填充

//PAD为填充量，对应上述图中末尾新增一列
__shared__ int tile[DIMY][DIMX+PAD]
unsigned int idx=threadIdx.y*blockDim.x+threadIdx.x;
// 按行将全局线程索引号，存入共享内存tile中
tile[threadIdx.y][threadIdx.x]=idx;
__syncthreads();
//按列读取tile中的元素，且不会产生存储体冲突
out[idx]=tile[threadIdx.x][threadIdx.y];

针对原始大小已知的共享内存，借助下图解释代码：

图一为想象图，可帮助我们逻辑上更好地理解，实际上图一所示的情形并不存在，GPU中实际的存储情况为图二所示。
我们可以假定图一和图二是等价的。想象中的图一，在GPU中自动对应为图二。

tile[threadIdx.y][threadIdx.x]=idx;

上句将线程号依次存入图一中所示的位置，填充padding列并不会有数值存入。（因为padding列所有坐标，不存在对应的threadIdx.x和threadIdx.y。 GPU内实际存储不会出现图一情况，而是图二所示。

out[idx]=tile[threadIdx.x][threadIdx.y];

读取时也是按图一去理解更容易，如果按照图二去看，记住padding元素没有被线程访问。

计算机中内存的存储是线性的，二维结构只是帮助我们逻辑上更好地理解

3.2 动态填充

//填充动态共享内存
extern __shared__ int tile[];
//求出全局线程号
unsigned int idx = threadIdx.y*blockDim.x+threadIdx.x;
//建立线程与内存之间的映射(非填充时）
//实现了坐标转置
unsigned int icol = idx % blockDim.y;
unsigned int irow = idx / blockDim.y;
//填充后的索引映射关系会发生改变
unsigned int row_idx=threadIdx.y*(PAD+blockDim.x)+threadIdx.x;
unsigned int col_idx=icol*(PAD+blockDim.x)+irow;
//tile[]是被填充的共享内存
tile[row_idx]=idx; //按行将idx存入共享内存，避免存储体冲突
__syncthreads();
out[idx]=tile[col_idx]; //按列读取, 填充避免了存储体冲突

row_idx将线程号对应到上述图一所示的位置，padding位显然没有任何线程访问
至此，成功实现矩阵转置且避免存储体冲突带来的性能下降

4 结语

填充的关键在于理解上文中图一和图二的联系：图一在计算机存储中不存在，但能够帮助我们更容易地理解填充过程。可以直接默认计算机将图一直接转化为图二，我们可以利用图一去辅助理解编写程序。
