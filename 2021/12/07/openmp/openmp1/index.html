OpenMP(1)——入门

2021-12-07- 2021-12-14
1k- 4m-
HPC-高性能计算OpenMP

本文主要内容：

  1. 使用OpenMP的通用步骤
  2. 任务task的概念
  3. 负载均衡

OpenMP并行一般包含两个步骤：

  1. 划分并行区域：指定并行运行的代码区域，并分配相应数量的线程
  2. 分配任务：将计算任务合理地分配到各线程

1 创建并行区域

通过parallel指令划分
常见有for, sections和tasks三种并行类型

1.1 for：循环结构

//下面语句等效为 划分并行区域+指定循环类型
#pragma omp parallel for

1.2 sections：非循环结构

#pragma omp parallel sections
{
    //多个section嵌套在sections内
    #pragma omp section{...} 
}

1.3 tasks：任务

不同于指令for和sections，task可动态定义任务
线程遇到task指令后开始创建任务，当创建的未分配线程的任务数量达到临界值时，停止创建任务并开始执行任务
一旦未分配的任务数量足够低时，线程再次开始创建任务

引入task的原因：
相较于for与section，task可并行实现不规则循环(如do while)与递归函数

//通过task递归求斐波那契数列
//由于b[i]的值取决于实际运行，parallel for无法并行下列代码
#pragma omp parallel for
for(int i=1; i<m; i+=b[i])
    {...}

//task实现递归
int fib(int n)
{
    int result,i,j;
    if (n<2) result=n;
    else
    {
        #pragma omp task shared(i) firstprivate(n) final(n<=10) mergeable
        i=fib(n-1);
        #pragma omp task shared(j) firstprivate(n) final(n<=10) mergeable
        j=fib(n-2);
        #pragma omp taskwait //确保计算f(n)时，f(n-1)与f(n-2)已完成计算
        result=i+j;
    }
    return result;
}

int main()
{
    #pragma omp parallel shared(n) 
    {
        ans=fib(n);
        #pragma omp single
        printf("the ans is %d\n", ans);
    }
}

2 分配线程

分配线程有两类方式：

  1. 静态分配：人为显式指定，线程数量固定
  2. 动态分配：系统按实际任务自行动态分配

注：线程数量过多但计算负载小时，切换线程的耗时高于执行任务耗时，导致并行效率低下

3 工作分配与负载平衡

在划分出并行区域后，需考虑如何将任务分配给不同的并行区域
基本原则：同一时刻，尽可能多的CPU核心同时工作，避免存在空闲核心(老资本家了😁)

3.1 为什么需要负载平衡

原因一：不同迭代次数的计算量不同
考虑以下循环

#pragma omp parallel for
for (int i=0; i<100000; i++)
    temp=i*i;

如果简单地将任务按数量均匀分配给多个线程，考虑负责i=99999的线程，需计算99999*99999，而分配到i=2的线程仅需计算2*2，此两者线程负载是显著不同的。

因此，直接按任务数量分配是不可行的，因为无法保证每个任务的计算量一致

原因二：划分的误差随核数增多而增大
假设有一个执行时间为16个时间单位的程序，如果划分误差为1个时间单位:
4核CPU上加速比16/(4+1)=3.2，是理想加速比的3.2/4=80%
16核CPU上加速比16/(1+1)=8，是理想加速比的8/16=50%

随着核心数增加，效率下降的比例更加明显

3.2 循环工作量的划分

  1. 静态负载平衡：人工划分工作，但无法得到任务实际运行耗时，因此无法保证实际运行时各个线程耗时相同
  2. 动态负载平衡：在任务执行过程中动态分配，人工设定调度策略后由编译器实现

策略分类：
static：静态调度，朴素地按任务数量划分
dynamic：动态调度，运行按实际情况使分配任务
guide：类似动态调度，但刚开始划分的任务块较大，随后逐渐减小

//schedule("类型","大小")
schedule(static,size)

动态调度会产生额外开销，但可以依据实际情况平衡负载
