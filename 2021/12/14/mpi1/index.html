简单搭建MPI

Posted on 2021-12-14 Edited on 2021-12-31 In HPC Views:

本文介绍通过SSH连接两台电脑，并成功运行MPI案例

MPI简介

MPI是一个通讯协议，可实现多台机器的并行计算
通俗来说，原本一台电脑的任务，现分配给多台电脑协作同步执行
众所周知，合作离不开沟通，MPI可以抽象理解成多台电脑间的沟通方式(如同人打电话、meeting等)

简易MPI搭建

实验环境：两台Linux虚拟机（同一局域网下两台Linux电脑亦可）

1 通过SSH连接两台电脑

注：两台电脑默认在同一局域网内，否则彼此间无法访问

1.1 SSH简介

  1. 连接时两台电脑分别为服务端和客户端，一般Ubuntu自带openssh-client，即客户端软件，充当服务端的电脑需执行apt install openssh-server安装openssh-server
  2. 客户端执行ssh-keygen生成密钥对，一直按enter的话，默认生成在路径./ssh内，可通过cd ~/.ssh进入
  3. 该目录下两个文件id_rsa、id_rsa.pub；前者为私钥，放在客户端，后者为公钥(public)，应放在服务器端，(我是用U盘拷贝到服务器的)

1.2 ssh配置文件

  1. cd /etc/ssh，移动到对应目录，运行ls，其中客户端配置文件为ssh_config, 服务器端为sshd_config
  2. 一般ssh_config使用默认配置即可

1.3 sshd的配置文件

  1. 使用vim打开sshd_config文件，按i进入编辑模式，找到以下对应设置并更改，注意删除对应的前面#
Port 22	端口号，服务器和客户端相同即可
ListenAddress 0.0.0.0	打开监听所有端口
PermitRootLogin yes	允许远程客户进入根权限
PublicAuthentication yes 打开密钥对认证模式
AuthorizedKeysFile .ssh/yourname.pub 名称为yourname.pub的公钥放置处
PasswordAuthentication yes 需要密码认证
PermitEmptyPasswords yes 不允许空密码
  1. esc, :x, 完成配置。注意x小写，大写的X是加密文件😄
  2. 重新启动ssh、sshd
server ssh restart
server sshd restart

1.4 生成公钥

以下操作两台电脑都需要依次执行：

  1. 添加相同用户：adduser mpi，su mpi切换至mpi用户
  2. 运行ssh-keygen -t rsa，在/.ssh目录下出现后缀名为.pub的文件，将其复制到另一台电脑的/.ssh目录下，注意文件名称需与sshd_config中设置一致
  3. 运行ifconfig，查看每台的电脑IP
  4. 运行vim /etc/hosts，给IP地址赋别名

    两台电脑hosts文件的设置相同
  5. 在IP地址为node1的电脑上，运行ssh node2尝试连接node2；反之亦然

补充：建议passwd -d mpi将mpi用户的密码删除(如果后续连接不上的话😄)

2 设置NFS挂载文件

两台电脑需区分服务器端和客户端，在服务器端创建共享文件夹，将需要共享的文件放于此文件夹内

mkdir ~/mpi/cloud

服务端与客户端两者文件位置应一致，方便后续运行mpi

2.1 NFS服务端配置

  1. sudo apt install nfs-kernel-server
  2. vim /etc/exports，在最后一行添加：
/home/mpi/cloud *(rw,sync,no_root_squash,no_subtree_check)
  1. 命令行运行exportfs -a
  2. 刷新服务 sudo service nfs-kernel-server restart

2.2 NFS客户端配置

  1. sudo apt-get install nfs-common
  2. mkdir cloud
  3. sudo mount -t nfs node1:/home/mpiuser/cloud ~/cloud
  4. df -h可查询是否成功挂载

此时在客户端应该能够看到服务端的共享文件

注：node1为自己在hosts文件中定义的服务端IP别名

3 运行mpirun

默认将编译成功的可执行文件(本案例文件名为test)置于共享文件夹cloud中，移动至该目录下执行：

mpirun -np 4 -host node1,node2 ./test

其中:

  • -np 4：表明调用4个进程
  • -host node1,node2：指定两台机器的IP地址
