OpenMP(2)——入门

Posted on 2021-12-14 In HPC Views:

本文主要内容：

  1. 常见的依赖关系(读后读、写后写、读后写)
  2. 执行不必要的工作实现分支并行

1 依赖关系

并行在同一时刻执行所有线程，需保证以任意顺序执行线程时，最终的结果一致
但如果线程之间存在依赖关系，不同顺序执行线程会导致不同的结果
因此，尽可能地消除依赖是并行的前提

2.1 循环依赖

2.1.1 读后写

读取元素A之后，下一次迭代会改变A的值
需保证先读后写，读到未被修改的初始值

for (int i=0; i<n; i++)
    a[i]=a[i+1];

//修改方式，创建一个数组a[]的副本
//副本在迭代中不会被修改，永远是原始值
for (int i=0; i<n; i++)
    a[i]=a_copy[i+1]

a[i]的值取决于a[i+1]，因此计算a[i]时需保证a[i+1]的值未被修改
对于a[i+1]而言，需“先读后写”，但并行时难以保证
可通过创建副本消除依赖

2.1.2 写后写

对元素A进行多次写入操作，元素A的最终值取决于最后一次写入操作

for (int i=0; i<n; i++)
{
    a[i]=b[i];
    a[i+1]=c[i];
    //修改建议: 提供副本存储每次迭代的a[i]
    a_copy[i]=b[i];
}

此处前两次迭代分别处理a[0],a[1]及a[1],a[2]
这两次迭代均对a[1]进行赋值，因此a[1]最终值取决于两次迭代的顺序，不能直接并行
同样的，可通过创建副本消除依赖

2.1.3 写后读

先更改元素A的值再读取，必须保证写操作先完成，否则读到A的旧值

for (int i=1; i<n; i++)
{
    b[i]=b[i]+a[i-1];
    a[i]=a[i]+c[i];
}
//重构代码以消除依赖
b[1]=b[1]+a[0];
for (int i=1; i<n-1; i++)
{
    a[i]=a[i]+c[i];
    b[i+1]=b[i+1]+a[i];
}
a[n]=a[n]+c[n]

上述修改前代码中，b[i]的值取决于a[i-1]的值，即第i次迭代依赖于第i-1次迭代(必须保证i-1次迭代先完成)
可通过重构代码消除依赖

2.2 任务依赖(并行if语句)

前后两个任务存在依赖关系，可通过执行不必要的工作实现并行化

int c=fun1(...);
if (c>c0) return c+fun2();
else return c;

fun1必须执行，fun2可能执行(取决于c是否大于c0)
设执行fun1需t1秒，执行fun2需t2秒，且实际执行fun2的几率为P
则串行总时间为t1+t2*P

改进思路：fun1与fun2同时并行执行，依据结果选择性输出
设同步开销时间为R，则并行代码运行时间为R+max(t1+t2)
当R+max(t1+t2) < t1+t2*P时，可提升效率

if分支难以并行，此处将判断语句延后进行，先计算再判断(俗称莽夫执行，干了再说😏)

int c1, c2;
#pragma omp parallel sections
{
    #pragma omp section
    {c1=fun1(...);}
    #pragma omp section
    {c2=fun2(...);};
}

if (c1>c2)
    return c1+c2;
else return c2;
