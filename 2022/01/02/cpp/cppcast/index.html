类型转换

Posted on 2022-01-02 In cpp Views:

最近读源码总遇到各种类型转换，今日来总结一下
本文涉及：

  • 上行与下行转换
  • static_cast, dynamic_cast和reinterpret_cast

上行与下行转换

  • 上行转换：将派生类指针或引用转化为基类表示
  • 下行转换(不安全)：将基类指针或引用转化为派生类表示

将基类理解为父亲，派生类理解为儿子；
下行转换即将基类指针(父亲)转化为派生类(儿子)，辈分降低了，故理解成下行

以下述代码为例，B为基类(base)，D为派生类(derived)

class B{
public:
  virtual void fun()
  {
    cout<<"access base class member: "<<m_b<<endl;
  }
private:
  int m_b = 11;
};

class D : public B {
public:
  void fun() override
  {
    cout<<"access derived class member: "<<m_d<<endl;
  }
  void dfun()
  {
    cout<<"this is derived class' special part "<<m_d<<endl;
  }

private:
  int m_d=99;
};

为什么下行转化不安全？

首先需理解派生类与基类的存储数据特点：
派生类不仅具有自己独特的部分，同时还包含基类部分，即派生类内存由基类数据与自身独有数据共同组成。
派生类指针ptrd，既可以访问派生类数据，亦可访问基类数据；
但基类指针ptrb不能访问派生类独有的数据

进行下行转化时，将基类指针ptrb转化为派生类类型的ptrd，表现为

B* ptrb = new b();
D* ptrd = new d();

D* static_ptrb_to_ptrd = static_cast<D*>(ptrb); //下行转换，不安全，用子类指向基类空间
static_ptrb_to_ptrd->fun();
static_ptrb_to_ptrd->dfun();

此处将ptrb类型由B* 转化为D*，但ptrb的内容并未发生变化
即转化后的指针static_ptrb_to_ptrd和ptrb指向同一块内存区域，该内存区域存放基类B的数据

但D*类型的指针static_ptrb_to_ptrd默认是可以访问派生类独有部分的
即static_ptrb_to_ptrd->dfun()若不在运行时检查，机器认为没有问题

那么问题来了，在机器实际存储中，基类不包含派生类的数据
虽然static_ptrb_to_ptrd->dfun()语法上没有问题，但机器内存中不包含dfun()部分
因此，下行转化不安全。

关键：在机器的实际存储中，基类空间不包含派生类部分，但派生类都包含基类部分

三种类型转换

static_cast

可用于替代C-style中的强制类型转换，static_cast仅在编译期检查，多用于非多态的转换。
上行转换安全，但由于缺少运行期检查，下行转换不安全。

dynamic_cast

多用于含虚函数的派生类之间转换，可进行运行时检查；
出现下行转化时报错，避免程序产生不安全行为

D* dynamic_ptrb_to_ptrd = dynamic_cast<D*>(ptrb1); //运行时检查，报错
dynamic_ptrb_to_ptrd->fun();
dynamic_ptrb_to_ptrd->dfun();

reinterpret_cast

最强大灵活，但需程序员自行保证安全性

reinterpret_cast字面意思为"重新解释"，语法上没有限制，甚至可将类指针转化为int指针

class A{
public:
  int m_a1=12;
  int m_a2=22;
};

int main()
{
  A* a=new A();
  cout<<a->m_a1<<" "<<a->m_a2<<endl;
  int* r = reinterpret_cast<int*>(a);
  
  cout <<*r<< endl;

  *r = 100;
  cout<<a->m_a1<<" "<<a->m_a2<<endl;
}

输出结果：

12 22
12
100 22

上述代码将内存中存放class A类型的数据，直接解释为int类型；
对r赋值为100后，改变原本存储空间数据，该存储空间原本存放class A的成员变量m_a1；
因此，改变r的值，直接影响了A的成员变量m_a1。

参考代码(自行注释不相关部分)：

#include <iostream>
using namespace std;

int char2int;

class B{
public:
  virtual void fun()
  {
    cout<<"access base class member: "<<m_b<<endl;
  }
private:
  int m_b = 11;
};

class D : public B {
public:
  void fun() override
  {
    cout<<"access derived class member: "<<m_d<<endl;
  }
  void dfun()
  {
    cout<<"this is derived class' special part "<<m_d<<endl;
  }

private:
  int m_d=99;
};

class A{
public:
  int m_a1=12;
  int m_a2=22;
};

int main()
{
  static_cast
  char a=10;
  cout<<char2int<<endl;
  char2int = static_cast<int>(a);
  // cout<<char2int<<endl;

  B* ptrb1 = new B();
  D* ptrd1 = new D();
  ptrb1->fun();
  ptrd1->fun();

  B* static_ptrd_to_ptrb = static_cast<B*>(ptrd1); //上行转换，安全，用基类指针指向子类空间
  static_ptrd_to_ptrb->fun();

  D* static_ptrb_to_ptrd = static_cast<D*>(ptrb1); //下行转换，不安全，用子类指向基类空间
  static_ptrb_to_ptrd->fun();
  static_ptrb_to_ptrd->dfun();

  // dynamic_cast
  D* dynamic_ptrb_to_ptrd = dynamic_cast<D*>(ptrb1); //运行时检查，报错
  dynamic_ptrb_to_ptrd->fun();
  dynamic_ptrb_to_ptrd->dfun();

  B* dynamic_ptrd_to_ptrb = dynamic_cast<B*>(ptrd1); //上行转换，安全
  dynamic_ptrd_to_ptrb->fun();

  //reinterpret_cast
  A* a=new A();
  cout<<a->m_a1<<" "<<a->m_a2<<endl;
  int* r = reinterpret_cast<int*>(a);

  cout << *r<< endl;

  *r = 100;
  cout<<a->m_a1<<" "<<a->m_a2<<endl;

  delete a;
  delete ptrd1;
  delete ptrb1;
}
