聊聊CRTP(奇异递归模板模式)

Posted on 2022-01-03 Edited on 2022-01-12 In cpp Views:

本文介绍CRTP的两个功能：

  1. 添加函数性(Adding functionality)
  2. 实现静态多态

CRTP

CRTP, Curiously Recurring Template Pattern，亦称奇异递归模板模式(这名字我就没记住过😬)
主要特征是：将派生类作为基类的模板参数。
通过这种奇怪的做法(很好奇第一个开发出这个套路的人是怎么想到的:) )，
主要得到了两种功能：Adding functionality与静态多态。

Adding functionality

假设B为基类,D为B的派生类；
借助CRTP可实现：在派生类D中定义某种特定函数时，D同时会自动获取额外的新函数功能。

例如：一般而言，operator++是借助operator+代码复用实现的。
那么，将operator++放在基类，则所有定义了operator+的派生类，均自动获取operator++的功能，实现了代码简化。

可见，CRTP的核心是将共同部分抽象到基类。

//1. Adding functionality
//将公共部分抽象到testUseCRTP中
//继承testUseCRTP的子类，只要定义getValue和setValue，即可获得square与setToOpposite方法
template <typename T>
struct testUseCRTP
{
    void square(){
        T& underlying=static_cast<T&>(*this);
        underlying.setValue(underlying.getValue()*underlying.getValue());
    };
    void setToOpposite(){
        T& underlying=static_cast<T&>(*this);
        underlying.setValue(-underlying.getValue());
    };
};

class A : public testUseCRTP<A>
{
public:
    A(double b){value=b;};
    ~A(){};
    double getValue() const{return value;};
    void setValue(double z){value=z;};
private:
    double value;
};

int main()
{
    A a(11);
    a.square();
    std::cout<<a.getValue()<<std::endl;
    a.setToOpposite();
    std::cout<<a.getValue()<<std::endl;
}

上述代码中，派生类A定义getValue()和setValue()两方法后，自动获取求平方square()和求相反数setToOpposite()两方法。

考虑一个问题：为什么使用CRTP，而不是non-member template functions呢？

  1. CRTP方案使类直接展示接口
  2. CRTP可在编译期报错，保证接口统一(一定程度上避免单词拼写错误)
  3. 模板函数不能方便定位在哪一个include文件中

静态多态

CRTP可使得类具有类似虚函数的功能，但没有虚函数调用时的开销（虚函数调用时，需通过虚函数指针查找虚函数表进行调用），但无法动态绑定。

虚函数：每个子类可定义函数将基类虚函数覆盖，实现不同的功能；抽象来看，CRTP也实现了不同子类的相同成员函数的特化，与虚函数得到的结果一致
虚函数表：编译器对于有虚函数的类，会维护一张虚函数表（虚表），对象前四个字节是指向虚表的指针。

//2. Static Interfaces
//CRTP的多态在编译期实现，属于静态多态
template<typename Child>
struct Base
{
    void implementation()
    {
        static_cast<Child*>(this)->implementation();
    }
};

struct Derived : Base<Derived>
{
    void implementation()
    {
        std::cerr<<"Derived implementation\n";
    }
};

int main()
{
    Derived d;
    d.implementation();
}

由上述代码可见，CRTP可实现类似虚函数的效果，同时避免虚函数调用时的开销（虚函数指针存储及查找）。
