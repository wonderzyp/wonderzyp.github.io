VTKm实现雪花分型案例

Posted on 2022-01-03 In cpp Views:

程序并行化需针对不同的硬件平台编写特定的代码，极大增加了编程工作量。
针对并行计算移植性差的这一痛点，VTK-m框架提供解决方案，实现了一份代码多平台通用。

0 VTKm框架简介

VTKm是Kitware公司(就是整出CMake的公司:smile:)旗下产品，VTKm官网称其为: A toolkit of SCIENTIFIC VISUALIZATION ALGORITHMS for emerging processor architectures。
虽然官方定位是可视化软件，但我们可将其用作程序中的一个模块，实现特定部分的并行化，并可保证程序在多平台通用。官方用户手册中也曾提到过这一点。

1 雪花分型概述

图(1) 分型迭代1次

图(2) 分型迭代2次
如上图所示，图(2)是在图(1)基础上迭代得到 以图(1)圈出的局部直线为例，迭代一次后变为图(2)中的局部四条直线 因此每一步迭代：**输入为一条直线，输出为4条直线**

2 VTKm框架实现

2.1 Run()

Run()函数接受参数numIterations，控制进行的迭代次数

VTKM_CONT static vtkm::cont::ArrayHandle<vtkm::Vec2f> Run(
    vtkm::IdComponent numIterations)
    {
        vtkm::cont::ArrayHandle<vtkm::Vec2f> points;

        points.Allocate(2);
        points.WritePortal().Set(0,vtkm::Vec2f(0.0f, 0.0f));
        points.WritePortal().Set(1, vtkm::Vec2f(1.0f, 0.0f));

        vtkm::cont::Invoker invoke;
        KochSnowflake::FractalWorklet worklet;

        for (vtkm::IdComponent i=0; i<numIterations; ++i)
        {
            vtkm::cont::ArrayHandle<vtkm::Vec2f> outPoints;
            invoke(worklet, points, outPoints);
            points=outPoints;
        }
        return points;
    }

上述代码使用数组points存储数据点，且初始化为{(0,0),(0,1)}
在for循环内部，以points数组为输入，输出至outpoints数组，并使用outpoints更新points数组

此处invoke函数的第一个参数worklet，控制分型的具体实现细节

invoke()函数可以理解为唤醒worklet，将points作为参数传递给worklet，并得到worklet的输出outpoints

2.2 worklet

VTKm框架内置部分worklet，本案例为了进一步阐述worklet原理，实现自定义worklet

2.2.1 worklet是一个functor

在C++中，如果定义类的时重载括号运算符，则可以向调用函数一样使用这个类

class absInt
{
    ...//省去构造析构等
    int operator()(int val){return val<0? -val : val;} 
}
absInt a;
int i=-1;
int ui=a(i); //a是一个absInt类，但可以像函数一样调用

2.2.2 worklet基本构成

struct FractalWorklet : vtkm::worklet::myWorklet
{
    using ControlSignature=void(SegmentsIn, SegmentsOut<4>);
    using ExecutionSignature=void(Transform, _2);
    using InputDomain=_1;

    template<typename SegmentsOutVecType>
    void operator()(const vtkm::exec::LineFractalTransform& transform, SegmentsOutVecType& segmentsOutVec) const
    {
        //...
    }
};

我们着重关注三个环节：
ControlSignature, ExecutionSignature和重载的operator()

(1) ControlSignature

ControlSignature的参数与前面的invoke函数参数一一对应

using ControlSignature=void(SegmentsIn, SegmentsOut<4>);
invoke(worklet, points, outPoints);

points对应SegmentsIn, outPoints对应SegmentsOut<4>

(2) ExecutionSignature

ExecutionSignature与ControlSignature参数相关联，并与operator()参数一一对应

using ControlSignature=void(SegmentsIn, SegmentsOut<4>);
using ExecutionSignature=void(Transform, _2);
void operator()(const vtkm::exec::LineFractalTransform& transform, SegmentsOutVecType& segmentsOutVec) const{...}

其中，_2表示对应ControlSignature的第二个参数SegmentsOut<4>
且(Transform, _2)与operator()参数类型严格匹配

(3) operator()

operator()是具体实现的环节，文章在后面会详细解释

至此，我们粗略了解VTKm使用逻辑，但依旧不清楚SegmentsIn, SegmentsOut<4>和Transform等意义。
本文将在接下来的自定义worklet环节具体介绍。

2.2.3 自定义worklet

本章主要介绍worklet的关键环节：
自定义ControlSignature Tags：SegmentsIn, SegmentsOut<4>
自定义ExecutionSignature Tags：Transform

(1) ControlSignature Tags

本案例自定义SegmentsIn和SegmentsOut两个ControlSignature Tags

//自定义ControlSignature Tags
struct SegmentsIn : vtkm::cont::arg::ControlSignatureTagBase
{
    using TypeCheckTag = vtkm::cont::arg::TypeCheckTag2DCoordinates;
    using TransportTag = vtkm::cont::arg::TransportTag2DLineSegmentsIn;
    using FetchTag = vtkm::exec::arg::FetchTag2DLineSegmentsIn;
};

template<vtkm::IdComponent NumSegments>
struct SegmentsOut : vtkm::cont::arg::ControlSignatureTagBase
{
    using TypeCheckTag = vtkm::cont::arg::TypeCheckTag2DCoordinates;
    using TransportTag = vtkm::cont::arg::TransportTag2DLineSegmentsOut<NumSegments>;
    using FetchTag = vtkm::exec::arg::FetchTagArrayDirectOut;
};

从上述代码可知，ControlSignature Tags包含三个步骤：

  1. 类型检查：TypeCheck
  2. 传输：TransportTag
  3. 分发：Fetch
TypeCheck

检查类型，只接受内部参数为vtkm::Vec2f_32和vtkm::Vec2f_64的arrayHandle。即数组内部元素必须为二维点，如points={(0,0),(1,0)}

Transport

VTKm将数据划分为两个环境：控制环境和执行环境
由于每个环境有自己的API，且彼此间不允许直接交互
我们定义的points和outpoint数组均为控制环境中的对象，而实际计算时需在执行环境中进行

Transport实现对象在两环境之间的转化

//2. Transport
template<typename ContObjectType, typename Device>
struct Transport<vtkm::cont::arg::TransportTag2DLineSegmentsIn,
				ContObjectType,
				Device>
{
	using GroupedArrayType = vtkm::cont::ArrayHandleGroupVec<ContObjectType, 2>;
	using ExecObjectType = typename GroupedArrayType::ReadPortalType;

	template<typename InputDomainType>
	VTKM_CONT ExecObjectType operator()(const ContObjectType& object,
		const InputDomainType&,
		vtkm::Id inputRange,
		vtkm::Id,
		vtkm::cont::Token& token) const
	{
		//GroupedArrayType将array中每两个点进行配对，以表示直线
		GroupedArrayType groupedArray(object);
		return groupedArray.PrepareForInput(Device{}, token);
	}
};

operator()的输入类型是控制环境对象ContObjectType，输出类型是执行环境对象ExecObjectType，实现对象转化。

同时，operator()将ContObject内部元素两两打包
ContObject内部元素是二维点，两点构成一条直线，因此ExecObject内部元素是线

例如：ContObject={(0,0),(0.5,0),(0.5,0),(1,0)}，表示四个点
则ExecObject={((0,0),(0.5,0), (0.5,0),(1,0))}，表示两条线

Fetch

Fetch将数组元素分配至不同线程中，实现不同线程处理不同数组元素

template<typename ExecObjectType>
struct Fetch<vtkm::exec::arg::FetchTag2DLineSegmentsIn, vtkm::exec::arg::AspectTagDefault, ExecObjectType>
{
	using ValueType = typename ExecObjectType::ValueType;
	VTKM_SUPPRESS_EXEC_WARNINGS
	template <typename ThreadIndicesType>
	VTKM_EXEC ValueType Load(const ThreadIndicesType& indices,
		const ExecObjectType& arrayPortal) const
	{
		return arrayPortal.Get(indices.GetInputIndex());
	}
};

重点关注return函数，各个线程通过各自的线程号，实现访问特定数组元素。

一个线程管一条局部直线

(2) ExecutionSignature Tags

本案例仅自定义一个ExecutionSignature Tag: Transform
涉及关键环节：

  1. 自定义线程类
  2. 自定义AspectTag
  3. 自定义LineFractalTransform类
//自定义ExecutionSignature Tags
struct Transform : vtkm::exec::arg::ExecutionSignatureTagBase
{
    static const vtkm::IdComponent INDEX = 1;
    using AspectTag = vtkm::exec::arg::AspectTagLineFractalTransform;
};

自定义的Transform需继承自ExecutionSignatureTagBase，并使用自定义的AspectTagLineFractalTransform

AspectTagLineFractalTransform

定义AspectTag旨在对传入的数据进行加工修饰

通过线程索引得到两个点GetPoint0和GetPoint1，返回类型是自定义的LineFractalTransform
可以理解为自定义的AspectTag对输入数据进行了加工，通过两点创建了一个类

注意此处执行环境是VTKM_EXEC

//核心代码
template<typename FetchTag, typename ExecObjectType>
struct Fetch<FetchTag, vtkm::exec::arg::AspectTagLineFractalTransform, ExecObjectType>
{
	using ValueType = LineFractalTransform;

	VTKM_SUPPRESS_EXEC_WARNINGS
	VTKM_EXEC ValueType Load(const vtkm::exec::arg::ThreadIndicesLineFractal& indices,
		const ExecObjectType&) const
	{
		return ValueType(indices.GetPoint0(), indices.GetPoint1());
	}
}

上文代码中涉及两个自定义的类：LineFractalTransform与ThreadIndicesLineFractal

ThreadIndicesLineFractal

线程类在创建时，依据自身线程号，读取输入数组的对应元素，并存储到私有成员变量中。

初始的线程仅线程号一个属性。
但创建线程类以后，通过设置成员变量，使每个线程持有更多的自身属性

//关键代码
class ThreadIndicesLineFractal : public vtkm::exec::arg::ThreadIndicesBasic
{
public:
    VTKM_EXEC ThreadIndicesLineFractal(vtkm::Id threadIndex, vtkm::Id inputIndex,
        vtkm::IdComponent visitIndex, vtkm::Id outputIndex, const InputPointPortal& inputPoints)
        : Superclass(threadIndex, inputIndex, visitIndex, outputIndex)
    {
        this->Point0 = inputPoints.Get(this->GetInputIndex())[0];
        this->Point1 = inputPoints.Get(this->GetInputIndex())[1];
    }

    VTKM_EXEC
        const CoordinateType& GetPoint0() const { return this->Point0; }

    VTKM_EXEC
        const CoordinateType& GetPoint1() const { return this->Point1; }

private:
    CoordinateType Point0;
    CoordinateType Point1;
};

自定义的线程类实现功能：
通过接口GetPoint(),用户可直接访问线程存储的两坐标点

LineFractalTransform

LineFractalTransform实现线段旋转功能：
局部一条直线迭代为四条折线时，有两条折线不再平行于原本直线，即与初始直线产生了一定夹角。我们通过两向量计算出偏移角

自定义的AspectTagLineFractalTransform返回类型是LineFractalTransform类
它的return函数调用LineFractalTransform的构造函数：通过输入的两点，计算出Offset, UAxis与VAxis来控制直线旋转角度，并存放于私有成员变量中

class LineFractalTransform
{
public:
    template<typename T>
    VTKM_EXEC LineFractalTransform(const vtkm::Vec<T, 2>& point0, const vtkm::Vec<T, 2>& point1)
    {
        this->Offset = point0;
        this->UAxis = point1 - point0;
        this->VAxis = vtkm::make_Vec(-this->UAxis[1], this->UAxis[0]);
    }

    template<typename T>
    VTKM_EXEC vtkm::Vec<T, 2> operator()(const vtkm::Vec<T, 2>& ppoint) const
    {
        vtkm::Vec2f ppointCast(ppoint);
        vtkm::Vec2f transform = ppointCast[0] * this->UAxis + ppointCast[1] * this->VAxis + this->Offset;
        return vtkm::Vec<T, 2>(transform);
    }

    template<typename T>
    VTKM_EXEC vtkm::Vec<T, 2> operator()(T x, T y) const
    {
        return (*this)(vtkm::Vec<T, 2>(x, y));
    }
private:
    vtkm::Vec2f Offset;
    vtkm::Vec2f UAxis;
    vtkm::Vec2f VAxis;
};

ExecutionSignature Tags后续作为worklet重载运算符的输入参数
即每条线段生成的LineFractalTransform会作为operator()一个参数

2.2.4 定义worklet的重载运算符

正如上文提及：worklet是一个functor，通过重载operator()类，可以像调用函数一样使用该类
因此，operator()的定义是实现任务的关键，本案例在operator()里生成四条新直线

using ControlSignature=void(SegmentsIn, SegmentsOut<4>);
using ExecutionSignature=void(Transform, _2);

template<typename SegmentsOutVecType>
void operator()(const vtkm::exec::LineFractalTransform& transform, SegmentsOutVecType& segmentsOutVec) const
{
    segmentsOutVec[0][0]=transform(0.00f, 0.00f);
    segmentsOutVec[0][1]=transform(0.33f, 0.00f);

    segmentsOutVec[1][0]=transform(0.33f, 0.00f);
    segmentsOutVec[1][1]=transform(0.50f, 0.29f);
    
    segmentsOutVec[2][0]=transform(0.50f, 0.29f);
    segmentsOutVec[2][1]=transform(0.67f, 0.00f);
    
    segmentsOutVec[3][0]=transform(0.67f, 0.00f);
    segmentsOutVec[3][1]=transform(1.00f, 0.00f);
}

operator()将transform作为输入，输出到segmentsOutVec
由ExecutionSignature可知，segmentsOutVec对应ControlSignature的第二个参数SegmentsOut<4>，进一步对应到outpoints数组

SegmentsOut<4>表示四条直线，每条直线的两端点通过transform类计算

transform类也是一个functor，因此可以像调用函数一样使用transform类

最后，将计算出的四条直线输出到outpoints数组中，完成一次迭代
