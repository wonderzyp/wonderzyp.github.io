聊聊移动语义和万能引用T&&

Posted on 2022-01-11 Edited on 2022-01-12 In cpp Views:

主要内容：

  1. 右值与移动语义
  2. 万能引用和引用折叠
  3. std::move, std::forward源码解析

写在前面：
本文将移动、右值、万能引用和完美转发尽可能简洁地阐述出来，可作为入门参考。
文章最后提供了进一步学习的参考资料。（C++真的是太令人痛苦了😄）

右值与移动语义

传统C++实现移动需要两步：先复制一份副本，再析构原始数据。
例如：想将一本书从学校移动到家中，传统C++解决思路是先买同一本相同的放在家中(复制)，再将学校的那本书扔了(销毁)，显然是非常浪费的。

为提高效率，C++11引入移动操作，避免先复制再析构的解决方案。

移动

以下面代码为例：

std::vector<int> foo(){
  std::vector<int> temp={1,2,3,4};
  return temp;
}

std::vector<int> v=foo();

按传统理解，foo()函数内部创建一个临时值temp并返回给v；
v在获取该临时值时，先复制一份副本，再将原始temp销毁，显然存在一个“复制-析构”的过程。
如果临时值temp相当大，则会产生大量开销影响性能。
所幸在C++11之后，编辑器将左值temp隐式右值转换，相当于static_cast<vector<int>&&>(temp)，并调用移动语义，避免“复制-析构”降低性能。

此外需注意，变量“移动”后，原先指针已变为nullptr，无法被再次访问：

int main() {
  std::string str = "zyp is handsome";
  std::vector<std::string> vec;
  vec.push_back(str);
  std::cout<<"str= "<<str<<endl;
  std::cout<<"using std::move"<<endl;

  vec.push_back(std::move(str));
  std::cout<<"str= "<<endl; //原始str处的值变为空
}

输出为:

str= zyp is handsome
using std::move
str=

移动通过避免“复制-析构”过程，减少开销并提升性能

接下来介绍右值引用，在C++中正是通过右值引用将移动和复制操作区分开。

右值

本章简单介绍左值和右值，以及右值在移动中的应用。(将亡值、纯右值可自行参考文末的参考资料)

  • 左值(lvalue)：可以被取地址的值，一般持续存在
  • 右值(rvalue)：表达式结束后不再存在

例如：int x=10，x是左值；20, 1+2等表达式结束后即消失的值称为右值。(右值没有在内存中开辟空间进行保存)

右值引用：T &&，可拿到一个将亡值，使临时值的生命周期得以延长，区分移动和复制操作。

下面以vector的push_back大致介绍右值引用在移动中的应用：

void push_back(const value_type& value){...};
void push_back(value_type&& value){...};

push_back通过重载函数为左值和右值分别提供一个独特版本的函数，如此可通过传入参数的类型将复制和移动操作分开：左值调用复制版本，右值调用移动版本。

但是，&&并不总是等价于右值引用。（是的，C++从不让人开心😄）欲理解这一点，需先了解万能引用的概念。

万能引用 T&&

在C++中，&&有时表示左值引用，有时表示右值引用。这同时也意味着：在特定情况下，&&既能绑定左值，又能绑定右值
正是这种灵活性，称&&为万能引用。
那么在什么情况下，&&可被视为万能引用呢？

当一个变量或参数被声明为T&&时，其中T为待推导类型，那么该变量或参数称为万能引用。（常见应用场景为模板函数或auto表达式）

万能引用依据初始化值类别不同，推到为不同的值类别，其推导规则如下：

  • 用左值初始化universal reference，则universal reference变成lvalue reference
  • 用右值初始化universal reference，则universal reference变为rvalue reference

这里介绍以下 值类别(value category) 和 值类型(value type) 的区别：

  • 值类别：左值、右值一类的概念
  • 值类型：int, double等

注意：值类别和值类型互不相干，这个规则最迷惑的就是右值引用本身是个左值。
以下面代码为例：

void fun(int&& arg){cout<<"rvaule reference"<<endl;}
void fun(int& arg){cout<<"lvalue reference"<<endl;}

int main() {
  int&& a=10;
  fun(a); //output: lvalue reference
  fun(10); //output: rvalue reference
}

代码中a的value type是int&&，即右值引用，但其调用重载函数的fun(int& arg)版本，可见其value category是左值
因此，值类别和值类型直接不存在必然联系。

另外需注意，仅当需要推导的情境下才能激活万能引用，类型已经明确的情况下无效：

int&& a=10;

int&& b=a; //错误，未激活万能引用，不能将左值和右值引用绑定
// int&& b=std::move(a); 正确，右值绑定至右值引用
auto&& c=a;// 激活万能引用，a是左值，此处auto&&推导为int&

总结：万能引用仅在推导情况下被激活，依据实际实例化参数的不同，既能推导为左值引用，又能推导为右值引用。

那么，万能引用是如何依据实例化参数进行推导的呢？
这涉及到C++的一个规定：引用折叠。

引用折叠

从一个简单案例开始：

template<typename T>
void fun(T&& arg);

int x=11; //x为左值
fun(10); //调用右值版本
fun(x); //调用左值版本

fun(10)和fun(x)分别调用以下实例：

void fun(int& arg); //左值版本
void fun(int&& arg); //右值版本

实际上，fun(x)调用模板函数时，有一个中间产物void fun(int & &&arg)；
但由于C++语法不允许指向引用的引用(reference to reference)，为解决上述的中间产物，C++规定了引用折叠法则：

  • rvalue reference to an rvalue reference 折叠为 rvalue reference
  • 其余情况折叠为lvalue reference

依据该法则：
void fun(int& &&arg)折叠为void fun(int& arg)
void fun(int&& &&arg)折叠为void fun(int&& arg
这也就解释了fun(10)和fun(x)调用不同版本的实现过程

再看下面一个案例:

//fun()函数在模板函数pass中调用
void fun( int const &arg ) { std::cout << "lvalue\n"; }
void fun( int && arg ) { std::cout << "rvalue\n"; }

template< typename T>
void pass(T&& arg) {
  std::cout << "using std::forward: ";
  fun( std::forward<T>(arg));
  std::cout << "using std::move: ";
  fun(std::move(arg));
  std::cout<<"using simple passing: ";
  fun(arg);
}

int main() {
  std::cout << "using rvalue 5:\n";
  pass(5); //传递右值
  cout<<endl;
  std::cout << "using lvalue x=5:\n";
  int x=5; //传递左值
  pass(x);
}

这里以pass(5)为例，解释程序运行过程：

  1. 5为右值，调用模板函数实例void pass(int&& &&arg)
  2. 由于引用折叠，实际为void pass(int&& arg)
  3. 在pass(int&& arg)中，arg是右值引用，但它是个左值
  4. std::forward可保留值的原有属性，int&&即调用fun(int&& arg)版本
  5. std::move强制转化为右值，一定调用fun(int&& arg)
  6. 简单传值fun(arg)，由于arg是左值，调用fun(int const &arg)

程序运行结果为：

//VScode -std=c++17

using rvalue 5:
using std::forward: rvalue
using std::move: rvalue
using simple passing: lvalue

using lvalue x=5:
using std::forward: lvalue
using std::move: rvalue
using simple passing: lvalue

这里显然有一个问题：对于pass(5)而言，明明刚开始传入的值是右值5，但最后间接调用fun函数的左值版本fun(int const& arg)
参数类型在传递的过程中发生了变化，原来是右值，却被无意识地转化为左值。
为解决这个问题，std::forward登场了 😄

功能与源码分析

  • std::forward：保证多次传递中value category始终不变(右值永远右值，左值永远左值)
  • std::move：强制转化为右值

下面分别介绍两者源码，但首先需了解std::remove_reference

std::remove_reference

std::remove_reference可去除T的引用部分，仅保留内存部分
如int&&和int&，均被转化为int
具体实现较简单，借助模板偏特化实现：

template<typename _Tp>
struct remove_reference
{ typedef _Tp   type; };

// 特化版本
template<typename _Tp>
struct remove_reference<_Tp&>
{ typedef _Tp   type; };

template<typename _Tp>
struct remove_reference<_Tp&&>
{ typedef _Tp   type; };

std::move

C++11使用std::move将参数无条件转化为右值

template<typename _Tp>  constexpr typename std::remove_reference<_Tp>::type&&  move(_Tp&& __t) noexcept  
{ 
  return static_cast<typename std::remove_reference<_Tp>::type&&>(__t); 
}

这个就更简单了，看它的返回类型std::remove_reference<_Tp>::type&&是一个右值引用，那么无论__t是_Tp&还是_Tp&&，均通过static_cast返回右值引用

std::forward

std::forward保证参数在传递过程中，其value category不会发生变化，这也被称为完美转发。
源码实现如下，需分别处理左值与右值：

//转发左值
template<typename _Tp>
constexpr _Tp&&
forward(typename std::remove_reference<_Tp>::type& __t) noexcept
{ return static_cast<_Tp&&>(__t); }

//转发右值
template<typename _Tp>
constexpr _Tp&&
forward(typename std::remove_reference<_Tp>::type&& __t) noexcept
{
  static_assert(!std::is_lvalue_reference<_Tp>::value, "template argument"
        " substituting _Tp is an lvalue reference type");
  return static_cast<_Tp&&>(__t);
}
  • 若__t为左值，调用第一个模板函数，_Tp&&会推导为_Tp& &&，进而折叠为_Tp&。由此实现接收左值，返回左值引用
  • 若__t为右值，匹配到第二个模板函数，_Tp&& &&折叠为_Tp&&。实现接收右值，返回右值引用。

总结：std::forward和std::move均借助static_cast实现；
前者保证value category在参数传递过程中不发生变化，后者实现强制转化为右值类型。
